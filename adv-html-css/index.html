new tags

section, header, footer, nav, article, aside, hgroup, audio, video

section

a thematic grouping of content, typically with a heading.

section vs. div

div has no semantic meaning

      
         <div class="secondary-wrapper">
           <section class="secondary">
             <section id="products">
             </section>
             <section id="workshops">
             </section>
           </section>
         </div>

header

a group of introductory or navigational aids.

footer

typically contains information about its section such as who wrote it, links to related documents, copyright data, and the like.

nav

a section of a page that links to other pages or to parts within the page: a section with navigation links.

article

a self-contained composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication.

aside

a section of a page that consists of content that is tangentially related to the content around the aside element, and which could be considered separate from that content.

hgroup

used to group a set of h1–h6 elements when the heading has multiple levels, such as subheadings, alternative titles, or taglines.

time

either a time on a 24 hour clock, or a precise date in the proleptic Gregorian calendar, optionally with a time and a time-zone offset.

<time datetime="2007-10-05">October 5</time>

Nesting

Nest selectors within their parents

      
        ul {
          margin: 2em 0;
        }

        ul li a {
          color: blue;
        }

        ul li a:hover {
          color: red;
        }

        ul li a span{
          text-align: right;
        }
      

      
        ul {
          margin: 2em 0;

          a {
            color: blue;

            &:hover {
              color: red;
            }

            span {
              text-align: right;
            }
          }
        }

Variables

Use the same color, same size, same fonts?

      
        $red: #AD141E;
        $body-font: "skolar-1","skolar-2", Georgia, serif;

        ul {
          li {
            font: 14px $body-font;
            color: $red;
          }
        }

Mixins

Mixins allow you to re-use whole chunks of CSS, properties or selectors. You can even pass variables through them.

      
      @mixin border-radius ($radii) {
  -webkit-border-radius: $radii;
     -moz-border-radius: $radii;
      -ms-border-radius: $radii;
       -o-border-radius: $radii;
          border-radius: $radii;
      }
      

      
        section {
          color: #000;
          @inculde border-radius(3px);
        }

Inheritance

Apply styles to selectors without adding more classs or IDs

      
        .clearfix:before, .clearfix:after {
          content: "\0020";
          display: block;
          height: 0;
          visibility: hidden; }

        .clearfix:after {
          clear: both; }

        .clearfix {
          zoom: 1; }
      

      
      section {
        color: #000;
        @include border-radius(3px);
        @extend .clearfix;
      }

Referencing Parent Selectors

Use & to reference parent selectors.

      
        a {
         font-weight: bold;
         text-decoration: none;

         &:hover { text-decoration: underline; }

         body.firefox & { font-weight: normal; }
       }

All variables for sass should be in this file. Make variables for fonts, colors, units, basic styles and more.

_base.scss

For your basic HTML styles

Top to bottom

When ordering selectors, order them from the top to the bottom of the DOM. DON’T JUST ADD THEM TO THE BOTTOM.

Alphabetic order

Have a consistent system when ordering CSS declarations, we use alphabetic.

      
        p {
          border-left:5px solid #ccc;
          color:#000;
          font-size:2em;
          font-style:italic;
          padding:1em 2em;
        }

Scss Spacing

One line always above a new selector selector and a space between attribute and value.

      
  blockquote {
    font-size: 2em;
    font-style: italic;
    padding: 1em 2em;
    color: #000;
    border-left: 5px solid #ccc;

    p {
      margin: 0;
    }

    cite {
      font-size: .5em;
      font-style: normal;
      font-weight: bold;
      color: #888;
    }
  }

CSS Layout

grid-width

Simple way to keep track the width 

      
        # The $gw-column and $gw-gutter variables must be defined in your base stylesheet to properly use the grid-width function.
        $gw-column: 100px;          // Column Width
        $gw-gutter: 40px;           // Gutter Width

        div {
          width: grid-width(4);     // returns 520px;
          margin-left: $gw-gutter;  // returns 40px;
        }

golden-ratio

Calulates golden ratio based on value.

      
        golden-ratio(14px,  1);    // returns: 22.652px
        golden-ratio(14px,  2);    // returns: 45.304px
        golden-ratio(14px, -1);    // returns: 8.653px

display: inline-block;

Element is placed inline but behaves as block

      box-shadow: h-shadow v-shadow blur spread color inset;
      @include inline-block;

display: table-cell;

Displays element same as <td>
